---
name: grill-me
description: Interview the user relentlessly about a feature or plan before writing any code. Use when the user wants to think through a design, plan a feature, or explore a problem space before implementation.
---
 
Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one by one.
 
If a question can be answered by exploring the codebase, explore the codebase instead of asking me.
 
Rules:
- Ask ONE question at a time — never batch multiple questions
- Start broad (what, why, who) then go deep (how, edge cases, errors)
- Challenge my assumptions — if something sounds vague, push back
- Track what's been decided so far — don't re-ask resolved questions
- When you've exhausted all branches, summarize the full plan and ask for confirmation
- Only after confirmation should you proceed to implementation or create a plan document
